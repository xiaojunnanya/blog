**前端开发中的性能优化对我们有哪些提升呢？**

- 用户体验：用户希望网站能够快速加载和相应，如果网站加载速度慢，用户可能会感到不耐烦而离开网站
- 减少服务器负载：优化前端性能可以减少服务器的负载，从而减少服务器的成本和维护成本
- 节省带宽：优化前端性能可以减少网站带宽的使用，从而减少带宽成本
- 提高转化率：快速加载的网站可以提高用户的满意度和转化率

当然，对于我们前端开发者来说，**用户体验**对我们来说是极为重要的。



## 首屏优化

- 预解析：添加`pre-parse`，预先解析DNS获取域名对应IP地址
- 预渲染：延后渲染无需立即打开的页面,确保使用时已渲染
  - `<link rel="dns-prefetch" href="http://www.google.com">`
- 懒加载：延后加载无需立即渲染的资源,进入区域时才加载
- 懒执行：延后执行无需立即处理的逻辑，等到使用时才执行
- 使用骨架屏

- 部分页面性能优化的
  - 图片懒加载、图片压缩、使用精灵图。
  - 服务端开启gzip压缩
  - 缓存



## 页面渲染性能优化

- 减少回流重绘
- 图片懒加载、图片压缩、使用精灵图。
  - 图片懒加载的原理是没有在可视区域的图片暂时不加载图片，等进入可视区域后在加载图片，这样可以减少初始页面加载的图片数量而提升页面加载速度
  - 图片懒加载在提升页面加载速度的同时也会伴随用户看其他未展示的图片时会有等待时间；图片加载显示会伴有布局抖动等问题

- 如果图片不能压缩，可以考虑图片分割，然后拼接
- 预加载：添加`pre-load`，延后加载无需立即用到的资源,确保使用时已加载
- 使用SSR
  - 服务端渲染：服务端返回 HTML 文件，客户端只需解析 HTML
  - 客户端渲染: 获取 HTML 文件，根据需要下载 JavaScript 文件，运行文件，生成 DOM，再渲染
- 服务端开启gzip压缩
- 浏览器缓存
  - 强缓存协商缓存等
- 组件按需引入
- 路由懒加载
- 组件懒加载
- 静态资源使用 CDN
  - 内容分发网络（CDN）是一组分布在多个不同地理位置的 Web 服务器
  - CDN是将源站内容分发至最接近用户的节点，使用户可就近取得所需内容，提高用户访问的响应速度和成功率
- DNS预解析
- 使用web workers优化长任务
- js加载使用acync和defer
- 骨架屏优化白屏时长
- 使用事件委托
- 长列表虚拟滚动
- 防抖节流
- 使用预加载
  - 预加载指的是将所需的资源提前请求加载到本地，这样后面在需要用到时就直接从缓存取资源。
